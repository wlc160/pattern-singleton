# pattern-singleton

设计模式之单例模式：

1、定义：
确保一个类在任何情况下都绝对只有一个实例，并提供一个全局访问点（创建型模式），例子：ServletContext、ServletConfig、ApplicationContext等

2、分类：

（1）饿汉式单例（在单例类首次加载时就创建实例）：

优点：

缺点：浪费内存空间

（2）懒汉式单例（被外部类调用时才创建实例）：

优点：

缺点：线程不安全（适中方案：双重检查锁）

（3）注册式单例（将每个实例都缓存到统一的容器中，用唯一标识获取实例）
优点：

缺点：线程不安全（适中方案：双重检查锁）

（4）ThreadLocal单例
优点：保证线程内部的全局唯一，且天生线程安全
